安装配置Git

1.安装依赖包

dnf install crul-devel expat-devel gettext-devel openssl-devel zlib-devel
dnf install gcc per-ExtUtils-MakeMaker

2.下载Git资源包，并解压编译安装

wget --no-ckeck-certificate https://www.kernel.org/pub/software/scm/git/git-2.36.1.tar.gz
tar -zxvf git-2.36.1.tar.gz -C /opt/src/ #/opt目录下的src目录已创建
make all prefix=/opt/git
make install prefix=/opt/git

3.配置Git

export PATH=/opt/git/bin/:$PATH

先配置环境变量 先在github上创建一个用户，然后在OpenEuler里面配置相应的用户名和邮箱，获取ssh密钥，将密钥添加到github，在github上创建一个远程仓库，在命令行输入命令关联远程仓库 在本地新建一个本地仓库，使用git相关命令就可以管理项目上传github啦

git init  #对本地仓库进行初始化
git add [项目名称]  #代码提交的暂存区
git commit -m "新建的一个项目"  #提交到本地仓库
git push -u origin master  #提交到远程仓库

使用docker和soket技术使客户端和服务器之间能进行信息传递

1.安装docker

dnf install docker
docker --version  #验证docker是否安装成功

2.在docker远程仓库下载tomcat镜像
因为后面要用到jdk，tomcat里面包含了jdk环境，所有这里选择了tomcat镜像

docker pull tomcat

3.运行镜像创建两个容器 这里的两个容器分别叫fwq（服务器端）和khd（客户端），再设置了端口映像，fwq的端口为2022,khd的端口为2023 4.查看服务器所在容器的IP地址

docker inspect fwq  #在宿主机上执行命令

5.将服务器端和客户端的代码程序文件拷贝到对应的容器镜像里面

docker cp git_space/SocketService.java fwq:/
docker cp git_space/SocketClient.java khd:/

SocketService.java是服务器端的源文件，SocketClient.java是客户端的源文件 服务器端

import java.io.*;
import java.net.*;

public class SocketService{
        public static void main(String[] args) throws IOException{
        SocketService socketService = new SocketService();
        socketService.oneServer();
        }
        public void oneServer(){
           try{
                   ServerSocket server=null;
                try{
                server=new ServerSocket(2022);
                System.out.println("Service enable Success");
        }catch(Exception e){
                System.out.println("No Listen:"+e);
        }
        Socket socket=null;
        try{
                socket=server.accept();
        }catch(Exception e){
                System.out.println("Error."+e);
        }
        String line;
        BufferedReader in =new BufferedReader(new InputStreamReader(socket.getInputStream()));
        PrintWriter writer=new PrintWriter(socket.getOutputStream());
        BufferedReader br=new BufferedReader(new InputStreamReader(System.in));
        System.out.println("Client:"+in.readLine());
        line=br.readLine();

        while(!line.equals("end")){
        writer.println(line);
        writer.flush();
        System.out.println("Service:"+line);
        System.out.println("Client:"+in.readLine());
        line=br.readLine();
        }

        writer.close();
        in.close();
        socket.close();
        server.close();
        }catch(Exception e){
                System.out.println("Error."+e);
        }

     }
}

6.开两个终端，分别进入两个容器，编译源文件，启动服务器和客户端

cd /
javac SocketService.java
java SocketService

使用Hugo静态网页生成器和Markdown语法编写一个静态网站

1.安装配置hugo

mkdir hugo
cd hugo
wget https://github.com/gohugoio/hugo/releases/download/v0.74.3/hugo_0.74.3_Linux-64bit.deb
dnf install dpkg
dpkg -i hugo_0.74.3_Linux-64bit.deb

2.新建一个站点 我在之前安装了hugo的目录下面新建了一个名为myblog的站点

hugo new site my_blog

3.下载一个主题 我下载了一个名为soho的主题

cd themes/  #在站点文件里面有个themes目录
git submodule add https://github.com/alexandrevicenzi/soho.git

4.在hugo目录下初始化git本地仓库

cd /my_blog
git init

5.在站点里面新建一篇文章

hugo new post/Linux课程大作业完成记录.md

我的文章名称为Linux课程大作业完成记录.md。 之后会在站点的content目录下产生一个post的目录，目录里面就是文章的.md文件，可以使用Markdown语法编写该文件生成文章 在站点的static目录下是存放一些静态文件，比如图片、js、css等 6.使用Markdown语法编写Linux课程大作业完成记录.md 文件 7.在本地根目录下生成public文件夹

hugo -D  #在my_blog目录下执行命令

6.创建一个tomcat服务器

docker run -d --name tomcatweb -p 8088:8080 tomcat

7.部署服务器 8.把静态网站部署到服务器上 8.新建一个仓库
